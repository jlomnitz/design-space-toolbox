Methods

The analysis I present here uses a new approach to analyze mathematical models composed of non-linear ordinary differential equations (ODEs). This approach, known as the system design space methodology, deconstructs a complex system into a finite number of non-linear sub-systems that represent dominant modes of operation for a particular system. The sub-system equations have a regular structure that are analytically tractable for a variety of system properties. The sub-systems are defined mathematically from the system equations and their behavior is representative of the full system within mathematically defined boundaries. The boundaries define a feasible region that is a convex polytope within parameter plus independent variable space.

The Design Space Toolbox V2, which I have developed as a major component of my PhD work, automates the difficult steps of the system design space method. It combines a computer algebra system and a variety of numerical methods to provide a rich framework for model analysis and visualization. This software is composed of a hierarchy of components: a C library performs the bulk of the analysis using multi-threaded parallel approaches; a SWIG-generated C-Python interface exposes the low-level functionality of the C library to Python; a Python Package provides a higher level interface that is object-oriented within a scientific programming environment; and a IPython-based widget provides a basic graphical user interface for wider accesibility.

Mathematical Models of Complex Phenomena¶

The analysis of complex systems does not begin with mathematical models; instead, they typically begin with a conceptual model that captures the key aspecs of a particular system. These features typically include interactions between different components and transiitons between different states. The conceptual model for the system I will analyze is shown in Figure 1. This conceptual model represents users interacting with the Twitch platform, grouped according to their activities i.e. starting a new session, viewing an online channel, browsing channels, viewing a live channel, successfully watching a video for more than 5 mins. These groups represent different states, and users can transition between states at particular rates that may vary as a function of a variety of independent variables that include the number of available channels, events such as tournaments, time of day, etc.

Mathematical and computational models can be formulated on the basis of these conceptual models that capture different aspects of the system behavior. For example, probabilistic models using Monte Carlo approaches model individual behaviors according to a particular set of rules. Alternatively, determininstic models can be formulated using differential equations that focus on the behavior of the system at the population-level, by modeling the average rates at which the transition between the states occur. These models can be useful to identify basic principles of a qualitative nature (i.e. principles that are independent of fine quantitative detail), through both numerical as well as analytical results.

In this analysis, I formulate a simple deterministic model defined by a system of differential equations that captures the change of each state variable as defined in the conceptual model. The right-hand side of each equation has a set of positive and negative rate-laws, where the rate-laws correspond to a specific mechanism or an aggregation of mechanisms by which users transition from one state to another, and corresponds to one of the arrows in the conceptual model. Each of these rate equations can be represented mathematically using a power-laws the advantage of being that they can capture non-linear behavior over wide-ranges and are self-similar and thus independent of scale.

Mathematical Model of User Sessions at a Population Level¶

The detailed mathematical model I will analyze in this notebook is defined by the following system of differential equations.

$\frac{dO}{dt} = \overbrace{\alpha_1 n \omega^{-1}}^{n \rightarrow O} + \overbrace{k_{31} \upsilon \omega^{-\frac{1}{2}}}^{\upsilon \rightarrow O} - \overbrace{k_{12} O}^{O \rightarrow b} - \overbrace{k_{13} O \omega^{\frac{1}{2}}}^{O \rightarrow \upsilon} - \overbrace{\beta_1 O}^{O \rightarrow \emptyset}$

$\frac{db}{dt} = \overbrace{k_{12} O}^{O \rightarrow b} + \overbrace{k_{32} \upsilon}^{\upsilon \rightarrow b} - \overbrace{k_{23} b}^{b \rightarrow \upsilon} - \overbrace{\beta_2 b}^{b \rightarrow \emptyset}$

$\frac{d\upsilon}{dt} = \overbrace{k_{23} b}^{b \rightarrow \upsilon} + \overbrace{\alpha_2 n \omega}^{n \rightarrow \upsilon} + \overbrace{k_{13} O \omega^{\frac{1}{2}}}^{O \rightarrow \upsilon} - \overbrace{k_{34} \upsilon}^{\upsilon \rightarrow S} - \overbrace{\beta_3 \upsilon}^{\upsilon \rightarrow \emptyset} - \overbrace{k_{32} \upsilon}^{\upsilon \rightarrow b} - \overbrace{k_{31} \upsilon \omega^{-\frac{1}{2}}}^{\upsilon \rightarrow O}$

$\frac{dS}{dt} = \overbrace{k_{34} \upsilon}^{\upsilon \rightarrow S} - \overbrace{k_{45} S}^{S \rightarrow n} - \overbrace{\beta_4 S}^{S \rightarrow \emptyset}$

$\frac{dn}{dt} = \overbrace{\alpha_n}^{\rightarrow n} + \overbrace{k_{45} S}^{S \rightarrow n} - \overbrace{\alpha_1 n \omega^{-1}}^{n \rightarrow O} - \overbrace{\alpha_2 n \omega}^{n \rightarrow \upsilon}$

where $O$ represents the number of people currently on an offline channel; $b$ represents the number of people browsing channels; $\upsilon$ represents the number of people actively viewing a video; $S$ represents the number of people that have 'succeded' by watching a video for 5 continuous minutes1; $n$ represents the number of users that are ready to initiate a session. The $\alpha_1$ and $\alpha_2$ parameters, with units of inverse minutes, represent the rate-constants for the transitions from users that are ready to initiate a session to an offline and live channel, respectively.

The $\omega$ parameter, with dimensionless units, represents the bias for a live channel relative to offline channels such that $\omega > 1$ represents a greater likelihood of a new session landing on a live channel, and $\omega < 1$ represents a greater likelihood of a new session landing on an offline channel.

The $\alpha_3$ parameter, with units of number of users per minute, represents the rate at which new users are initiating a session. the $k_{ij}$ parameters, with units of inverse minutes, represent the rate-constants for the transition from the group of users represented in the $i$-th equation to the group of users represented by the $j$-th equation (e.g. $k_{12}$ is the rate constant for the transition from users on offline channels to browsing pages).

The $\beta_i$ parameters, with units of inverse minutes, represent the rate-constants for failure for the group represented by the $i$-th equation. In this model, I assume that the highest likelihood of users failing is when on offline channels - thus, $\beta_1 > \beta_i$ for $i = {2, 3, 4}$.

The $\sigma$ < 1 parameter, with dimensionless units, represents the frequency at which users on live channels succeed, such that $\sigma\frac{1}{5}$ is the observed rate-constant for success, which by definition is at most $1/5$ min$^{-1}$.

The observed rate constant is expected to be less than the maximal value because live users may watch multiple live channels before succeeding. The overbrace is used to indicate the corresponding transition in the conceptual model, shown in Figure 1.

The design space method, as mentioned previously, decomposes this complex system into a finite set of non-linear sub-systems. Each sub-systems represents a dominant mode of the system with a characteristic behavior. Associated with each dominant mode of the system there is a convex $m$-dimensional polytope, where $m$ is the number of parameters plus independent variables of the system. By applying methods from linear programming and computational geometry, the validity of each of the qualitatively distinct behaviors can be determined without sampling values for the parameters. Furthermore, each of these behaviors can be analyzed independently from one another, thus the computations can be performed in parallel.

The conceptual model, shown in Figure 1, divides users into different groups according to their current activities. As users interact with the Twitch platform, their activity changes and thus they transition from one state to another, as shown by the arrows in Figure 1.

A detailed description of this model, by using an individual user as an example, is as follows.

We begin with a user that becomes available to initiate a Twitch session. In our model, this user is in state n and transitioned to n externally from the system through "→ n". Once available to initiate a session, there are 2 possibilities: they can (1) start the session on a channel that is offline, thus they transition to the O state through the "$n$ → $O$" arrow; or (2) start a session on a channel that is currently live, thus they transition to the $\upsilon$ state through "n → $\upsilon$" arrow. An assumption of the model is that the user cannot start the session browsing videos. Users on offline channels, represented by $O$, can either fail and navigate away from the Twitch website, shown by the diagonal arrow pointing outwards from the system; begin browsing available channels, shown by the "$O$ → $b$" arrow; or go directly to a live channel, shown by the "$O$ → $\upsilon$" arrow. Similarly, users on live channels, represented by $\upsilon$, can fail; go back to browsing videos, shown by the "$\upsilon$ → $b$" arrow; follow a direct link to an offline channel, shown by the "$\upsilon$ → $O$" arrow; or succesfully complete a session, shown by the "$\upsilon$ → $S$" arrow. Users that are browsing videos are assumed to either fail or browse to a live channel, shown by the "$b$ → $\upsilon$" arrow. Once a user has had a succesful session, represented by $S$, they can either become available to initiate a become new session "$S$ → $n$" arrow, or fail.

A mathematical model for this conceptual model can then be proposed for rigorous analysis. However, models of different types can be used to obtain different types of understanding. For example, one could model the behavior of a individual user using a probabilistic model and a Monte Carlo approach. Repeating the experiment may provide information regarding different trends such as population averages. Another alternative is to use deterministic models, such as those defined by systems of differential equations, that purposefully model the average behavior of the users. The advantage of the deterministic model, as shown in this analysis, is that we can find conditions among the parameters, variables and rates of the system to identify principles through numerical and analytical methods.

def create_table_subcases(cases, table_number=0, title='', caption=''):
    table_body = ''
    for index,case in enumerate(cases):
        styles = [';'.join(style[:2+(index % 3 if i == 0 else 0)]) for i in xrange(3)]
        data = ['<b>'+case.case_number+'</b>',
                case.ssystem.log_gain('S', 'k12'), 
                case.ssystem.log_gain('S', 'om')]
        color_style = [colors[i]
                       for datum in data[1:] 
                       for i,test in enumerate([datum < 0, datum == 0, datum > 0])
                       if test is True]
        data = data + [colors[3]] + color_style
        row = ''.join([row_template[i].replace('$style_'+str(i), styles[i]) for i in xrange(3)])
        row = row.format(*data)
        if index % 3 == 0:
            row = '<tr style="border:0;text-align:center;">'+row
        elif index % 3 == 2:
            row = row+'</tr>'
        table_body += row
    table = html_preamble.replace('$table_body', table_body)
    table = table.replace('$table_number', 'Table '+str(table_number))
    table = table.replace('$title', title)
    table = table.replace('$caption', caption)
    return table

Close inspection of Table 1 shows some things we might expect. Increasing the bias for live channels, $\omega$, cannot decrease the number of succesful views (i.e. $\omega$ is either uncoupled from $S$, meaning a change in $\omega$ does not change $S$, or directly coupled, meaning an increase in $\omega$ elicits an increase in $S$). However, increasing the rate at which users transition from offline channels to browsing pages, represented by $k_{12}$, can increase the number of users with succesful views, but and can also decrease the number of succesful views.

This is counter intuitive because of two reasons: (1) the number of users browsing pages should increase proportionally with the decrease in users on offline channels as we increase $k_{12}$; and (2) the rate-constant for failure for users on offline channels is greater than the rate-constant for failure for users that are browsing pages.
